## 1. N-gram 概率计算
**题目**：自然语言处理中，语言模型经历了从统计到神经网络的模型演进。请使用本章提供的迷你语料库（datawhale agent learns, datawhale agent works），计算句子 agent works 在 Bigram 模型下的概率。

**解答**：
语料库：`datawhale agent learns`, `datawhale agent works`
总词数：6 (datawhale: 2次, agent: 2次, learns: 1次, works: 1次)
在 Bigram 模型中，句子 "agent works" 的近似概率为：
$P(\text{agent works}) \approx P(\text{agent}) \times P(\text{works}|\text{agent})$
1. 计算 $P(\text{agent}) = \frac{\text{Count}(\text{agent})}{\text{总词数}} = \frac{2}{6} \approx 0.333$
2. 计算条件概率 $P(\text{works}|\text{agent}) = \frac{\text{Count}(\text{agent works})}{\text{Count}(\text{agent})} = \frac{1}{2} = 0.5$
3. $P(\text{agent works}) \approx 0.333 \times 0.5 \approx 0.167$

## 2. 马尔可夫假设与 N-gram 局限性
**题目**：N-gram模型的核心假设是马尔可夫假设。请解释这个假设的含义，以及N-gram模型存在哪些根本性局限？

**解答**：
- **马尔可夫假设**：一个词的出现概率只与它前面有限的 $n-1$ 个词有关，而不必回溯该词在句子中的全部历史。
- **根本性局限**：
  1. **数据稀疏性 (Sparsity)**：如果一个词序列从未在训练语料库中出现过，其概率估计就为 0，这在自然语言极其丰富的组合中非常不合理。
  2. **泛化能力差**：模型将词视为孤立的符号，无法理解词与词之间的语义相似性（例如模型不知道 agent 和 robot 在特定语境下含义相近）。

## 3. RNN/LSTM 与 Transformer 的克服方式
**题目**：神经网络语言模型（RNN/LSTM）和Transformer分别是如何克服N-gram模型局限的？它们各自的优势是什么？

**解答**：
- **RNN/LSTM**：
  - **克服方式**：引入了连续的**词嵌入（Word Embedding）**解决了泛化能力差的问题；通过**隐藏状态（短期记忆）**打破了固定上下文窗口的限制。
  - **优势**：能够处理任意长度的序列数据，理解词汇的语义相似度，捕捉相对长距离的依赖关系（尤其是 LSTM 的门控机制）。
- **Transformer**：
  - **克服方式**：彻底抛弃循环结构，完全依赖**注意力机制（Attention）**来动态捕捉序列内任意两个词汇之间的全局依赖关系。
  - **优势**：从根本上解决了 RNN 必须串行计算的问题，实现了**大规模的并行计算**，极大提升了模型训练速度和规模上限。

## 4. Transformer 核心机制
**题目**：Transformer架构是现代大语言模型的基础。其中：
- 自注意力机制（Self-Attention）的核心思想是什么？为什么Transformer能够并行处理序列，而RNN必须串行处理？
- 位置编码（Positional Encoding）在其中起什么作用？

**解答**：
- **自注意力机制核心思想**：允许模型在处理序列中的每一个词时，同时计算它与句子中所有其他词的相关性（分配注意力权重），从而动态融合全局上下文信息。
- **并行与串行**：RNN 依赖于上一个时间步的隐藏状态才能计算当前时间步，导致必须按顺序串行处理；而 Transformer 的自注意力机制通过矩阵乘法，一次性对序列中的所有位置进行相互计算，天然支持并行。
- **位置编码作用**：自注意力机制本身不包含词的顺序信息（打乱句子顺序计算结果一样）。位置编码为每个词元的向量加上代表绝对和相对位置的固定数值向量，让模型能够感知序列中词汇的先后顺序。

## 5. Decoder-Only 架构
**题目**：Decoder-Only架构与完整的Encoder-Decoder架构有什么区别？为什么现在主流的大语言模型都采用Decoder-Only架构？

**解答**：
- **区别**：Encoder-Decoder（如原版 Transformer）是“先理解后生成”，编码器读取全部上下文，解码器基于其输出生成结果；Decoder-Only 抛弃了编码器，采用**自回归（Autoregressive）**模式，只利用掩码自注意力机制，通过不断“预测下一个词”来生成文本。
- **主流原因**：
  1. **训练目标统一**：唯一的任务就是“预测下一个词”，极度契合海量无标注文本的自监督预训练。
  2. **结构简单、易于扩展**：组件少，更容易堆叠参数和实现规模化（Scaling Laws）。
  3. **天然适合生成任务**：对话、代码生成、续写等任务本质上都是序列生成。

## 6. 分词算法与 BPE
**题目**：文本子词分词算法是大语言模型的一项关键技术。为什么不能直接以"字符"或"单词"作为模型的输入单元？BPE算法解决了什么问题？

**解答**：
- **为什么不用单词/字符**：
  - 按单词分词：会导致词表极其庞大，且经常遇到未登录词（OOV），也无法处理词缀变化（如 look, looking 被视为毫无关联的词）。
  - 按字符分词：词表极小且无 OOV 问题，但单个字符缺乏独立语义，导致序列变得非常长，降低了模型学习和推理效率。
- **BPE 解决的问题**：字节对编码（BPE）是一种子词分词算法。它通过统计语料库中相邻词元对的频率，迭代合并最高频的组合。它完美平衡了词表大小和语义表达，常见的词作为整体保留，罕见的词被拆分为有意义的子词，从而彻底解决了 OOV 问题，并提高了分词效率。

## 7. 本地部署与提示工程实践
**题目**：按照指导部署轻量级开源模型，调整采样参数并观察影响；选择任务设计对比不同的提示策略（Zero-shot, Few-shot, CoT）。

**解答**：

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

def evaluate_prompt_strategies(prompt_type: str, text: str) -> str:
    """
    根据不同的提示策略生成最终的提示词。

        包装后的完整提示词字符串

        return f"请判断以下文本的情感倾向。请一步一步地思考（Let's think step by step），先分析句子中的情感词汇，再给出最终结论（正面/负面/中性）。\n文本：{text}\n分析与情感："
```

## 8. 开源与闭源模型选择
**题目**：从性能、成本、可控性、隐私等维度比较闭源模型和开源模型。构建企业级客服智能体会选择哪种类型的模型？

**解答**：
- **维度对比**：
  - **性能**：闭源模型（如 GPT-4）通常代表行业天花板，通用能力极强；开源模型（如 Llama 3）在中等参数量下性价比极高。
  - **成本**：闭源按 Token 计费，前期投入低；开源模型免费，但需要自行承担昂贵的算力硬件（GPU）和运维成本。
  - **可控性**：开源模型完全自主可控，支持深度微调（SFT）；闭源模型受限于 API 接口及服务商的底层更新。
  - **隐私**：开源模型可实现 100% 物理隔离与本地部署，隐私最高；闭源必须将数据传至云端，存在数据泄露风险。
- **企业级客服智能体选型建议**：
  - **推荐选择**：**基于开源模型（如 Qwen 或 Llama 系列）进行本地部署与微调**。
  - **考虑因素**：客服业务往往涉及大量企业内部产品数据和用户隐私（如订单号、联系方式），数据合规性是红线；同时，客服业务具备“领域垂直”特性，利用企业历史客服聊天记录对开源模型进行微调，能在特定场景下达到甚至超越闭源大模型的效果，长期的推理成本也可控。

## 9. 模型幻觉与缓解方法
**题目**：选择一种缓解幻觉的方法，说明其工作原理和适用场景。调研前沿的研究和论文，是否还有其他的缓解方法？

**解答**：
- **检索增强生成 (RAG)**：
  - **工作原理**：在模型生成回答之前，先将用户的 query 转化为向量，从外部构建的知识库（如企业文档库）中检索出最相关的片段。然后将这些真实片段作为上下文拼接到提示词中，强制模型基于“开卷”的内容进行回答。
  - **适用场景**：垂直领域的知识问答（如法律、医疗）、企业内部文档助手、时效性要求极高的场景。
- **前沿缓解幻觉的其他方法**：
  1. **自我纠错与验证 (Self-Correction/Self-Reflection)**：让大模型在输出后，自己扮演评估者，利用特殊的 Prompt 对刚生成的答案进行多轮事实核查与修改（如 Chain-of-Verification, CoVe）。
  2. **知识编辑 (Knowledge Editing)**：直接定位模型权重中存储错误知识的特定神经元，通过 ROME 等算法对其进行定向修改，从底层抹除幻觉根源。
  3. **基于人类反馈的直接偏好优化 (DPO / RLHF for Factuality)**：在对齐阶段专门构建包含“诚实回答”和“幻觉回答”的偏好数据集，训练模型主动拒绝回答不知道的问题。

## 10. 论文辅助阅读智能体设计
**题目**：设计论文辅助阅读智能体。基座模型选择？提示词设计？上下文窗口限制解决？如何保证严谨客观？

**解答**：
- **基座模型选择**：选择具有**超大上下文窗口**和**强逻辑推理能力**的模型（如 Claude 3.5 Sonnet 或 Gemini 1.5 Pro）。
  - **考虑因素**：论文通常长达数万字，模型必须具备不遗忘的长文本处理能力（Needle In A Haystack 表现好）；同时提取多篇论文观点需要极强的跨文档推理与总结能力。
- **提示词设计**：
  - 采用**角色扮演**和**结构化输出**约束。例如：“你是一位顶尖的人工智能领域研究员。请仔细阅读以下论文内容，并严格按以下结构输出：1. 核心创新点 2. 方法论 3. 实验结论。如果文中未提及某点，请直接回答‘原文未提及’，绝不可捏造。”
- **上下文窗口限制解决策略**：
  1. **采用 RAG 架构**：将长论文按章节（Abstract, Method, Experiments）切块并向量化，当用户提问时，仅检索相关章节作为上下文喂给模型。
  2. **Map-Reduce 总结法**：先让模型逐段生成小摘要，最后再对所有小摘要进行一次全局合并总结。
- **确保严谨客观（Grounding）设计**：
  1. **强制溯源 (Citation Requirement)**：在系统提示词中规定，生成的每一条结论必须附带原文的精确引用（如具体页码、章节名或段落截图链接）。
  2. **引入多智能体交叉验证 (Multi-Agent Debate)**：一个 Agent 负责提取和总结，另一个“批评家 Agent”负责对照原文逐句挑刺，如果发现生成的结论在原文中找不到依据，则打回重做。
