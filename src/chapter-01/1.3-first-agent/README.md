# 1.3 动手体验：5 分钟实现第一个智能体

## 任务背景
在前面的小节，我们学习了智能体的任务环境、核心运行机制以及 Thought-Action-Observation 交互范式。
本案例的目标是构建一个能处理分步任务的智能旅行助手。

## 任务目标
**用户指令**："你好，请帮我查询一下今天北京的天气，然后根据天气推荐一个合适的旅游景点。"

**核心逻辑**：
1.  Thought（思考）：分析用户需求，需要先查询天气。
2.  Action（行动）：调用天气查询工具。
3.  Observation（观察）：获取天气信息（例如：晴天）。
4.  Thought（思考）：基于天气信息，决定推荐景点。
5.  Action（行动）：调用景点推荐工具。
6.  Observation（观察）：获取景点信息（例如：故宫）。
7.  Final Answer（最终回答）：向用户汇报结果。

## 核心概念
- **Thought-Action-Observation 循环**：智能体通过不断的思考、行动和观察来逐步解决问题。
- **Tools（工具）**：智能体可以调用的外部函数（如查询天气、推荐景点）。

## 如何运行
1.  安装依赖（如果需要）：`pip install openai` (本示例可能使用模拟数据或 API)
2.  运行代码：`python simple_agent.py`
